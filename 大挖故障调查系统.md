## 一、系统总体目标

构建一个面向大挖公司内部员工的实时故障调查系统，实现故障记录的可视化、可追踪、可协作、可分析，提升故障响应与闭环效率。

------

## 二、功能需求清单

### 1. 用户认证与登录模块

| ID   | 功能点       | 详细说明                                                     |
| :--- | :----------- | :----------------------------------------------------------- |
| A1   | 飞书授权登录 | 仅允许大挖公司内部员工通过飞书OAuth授权登录，获取用户身份信息（姓名、部门、工号）。 |
| A2   | 首次登录引导 | 首次登录时，系统自动记录用户信息，并分配默认角色（如“普通成员”），管理员可后续调整。 |
| A3   | 登录态保持   | 支持7天内自动登录（基于JWT或类似机制），登出后清除凭证。     |

### 2. 权限系统

| ID   | 功能点       | 详细说明                                                     |
| :--- | :----------- | :----------------------------------------------------------- |
| B1   | 角色定义     | 预设三种角色：**管理员**（所有权限）、**工程师**（可编辑/新增故障，不可删除）、**观察员**（仅查看和导出）。 |
| B2   | 权限控制     | - **删除故障**：仅管理员 - **修改故障**：管理员 + 责任人/调查人本人（可修改自己负责的故障） - **新增故障**：管理员 + 工程师 - **查看/导出**：所有角色 |
| B3   | 权限变更日志 | 角色变更操作需记录到日志系统，包括操作人、时间、变更前后角色。 |

### 3. 日志系统（故障修改记录）

| ID   | 功能点       | 详细说明                                                     |
| :--- | :----------- | :----------------------------------------------------------- |
| C1   | 自动记录操作 | 对故障记录的**新增、修改、删除**均记录日志，包括：操作人、时间、操作类型、故障唯一标识、修改前后的字段内容（对于修改）。 |
| C2   | 日志查询     | 管理员可按故障ID、操作人、时间范围查询操作历史，支持导出为CSV。 |
| C3   | 日志存储     | 日志保留至少2年，支持分页展示。                              |

### 4. 当日故障可视化展示（轻重主次）

| ID   | 功能点           | 详细说明                                                     |
| :--- | :--------------- | :----------------------------------------------------------- |
| D1   | 当日故障定义     | 早会日期 = 当天的故障记录，以及最近3天内未闭环的故障（需配置）。 |
| D2   | 轻重主次划分规则 | 依据表格中的“故障风险评估”字段： - **高（紧急）**：有批量风险 - **中（重要）**：零星偶发 - **低（一般）**：首发 另可增加“未对策”标记为“待处理”。 |
| D3   | 可视化卡片/列表  | 按优先级分组展示：顶部展示紧急故障卡片，其次重要，最后一般。每个卡片显示：机号、机型、故障描述、责任人、故障风险评估、是否闭环。 |
| D4   | 颜色标记         | 紧急-红色背景，重要-黄色，一般-灰色，未闭环-闪烁/加粗边框。  |
| D5   | 点击详情         | 点击任意故障卡片，侧拉抽屉展示该故障的完整信息（所有表格字段+照片）。 |
| D6   | 实时刷新         | 页面每5分钟自动刷新，或用户手动刷新。                        |

### 5. 后台故障管理（增、删、改、进度更新）

| ID   | 功能点   | 详细说明                                                     |
| :--- | :------- | :----------------------------------------------------------- |
| E1   | 新增故障 | 表单包含表格中所有字段（见附录1），其中“早会日期”默认为当天，“故障描述”必填，“机型/机号”必填，支持上传最多4张照片（JPG/PNG）。 |
| E2   | 修改故障 | 支持对已有记录的任意字段修改（权限受B2限制），修改后自动更新“最后修改时间”并写入日志。 |
| E3   | 删除故障 | 管理员可删除故障，需二次确认，软删除（标记删除，数据保留用于统计）。 |
| E4   | 进度更新 | 专门提供“更新进展”按钮，用于修改“临时对策”、“长久对策及进展”、“是否闭环”字段，并可追加备注（带时间戳）。 |
| E5   | 批量导入 | 支持Excel批量导入故障（按模板），避免手工逐条新增，需校验必填字段。 |

### 6. 历史故障查询

| ID   | 功能点       | 详细说明                                                     |
| :--- | :----------- | :----------------------------------------------------------- |
| F1   | 查询字段     | 支持按以下表头字段进行**精确或模糊查询**： - 早会日期（范围） - 分类（国内故障/国际故障/国内巡检等） - 机型（SY750H, SY980H等） - 机号（精确或后几位） - 服务工程师 - 是否对策（对策前/对策后） - 故障风险评估 - 是否闭环 - 整机生产地（昆山/重庆） - 故障部位3（关键词） |
| F2   | 组合查询     | 多个条件可同时使用，逻辑为“AND”，支持保存常用查询条件。      |
| F3   | 查询结果展示 | 以表格形式展示，表头可自定义显示/隐藏列，支持排序（按早会日期倒序为默认）。 |
| F4   | 分页         | 每页默认20条，可选50、100条。                                |

### 7. 报表导出

| ID   | 功能点   | 详细说明                                                     |
| :--- | :------- | :----------------------------------------------------------- |
| G1   | 导出范围 | 基于当前查询结果（筛选后的数据）导出为Excel文件（.xlsx）。   |
| G2   | 导出字段 | 可选择导出所有字段或自定义字段（提供勾选框）。               |
| G3   | 导出格式 | 保留原表格样式（日期格式、文本换行），照片不嵌入，仅导出照片文件名或链接。 |
| G4   | 导出权限 | 所有登录用户均可导出，但需记录导出日志（谁、何时、导出条件）。 |

### 8. 额外建议功能（您未明确但实用）

| ID   | 功能点         | 详细说明                                                     |
| :--- | :------------- | :----------------------------------------------------------- |
| H1   | 故障趋势看板   | 按月份展示故障数量折线图，可筛选机型、分类，显示故障率（故障数/保有量，需接入保有量数据）。 |
| H2   | 责任人待办看板 | 每个工程师登录后，看到自己作为“回复责任人”或“调查人”的未闭环故障列表，支持一键更新进度。 |
| H3   | 通知提醒       | 当故障被分配给自己、或故障状态变为“有批量风险”时，通过飞书消息卡片通知责任人。 |
| H4   | 故障照片管理   | 支持在查看详情时放大照片、多图轮播，且照片可替换或删除。     |
| H5   | 数据字典维护   | 管理员可维护“分类”、“机型”、“故障风险评估”等下拉选项的值。   |
| H6   | 操作引导与帮助 | 提供简单的新手引导（首次登录时），以及各页面的帮助入口。     |

## 四、非功能性需求

| 类别       | 要求                                       |
| :--------- | :----------------------------------------- |
| 响应时间   | 列表查询 < 2秒，导出 < 5秒（500条以内）    |
| 并发支持   | 至少支持50人同时在线操作                   |
| 数据备份   | 每日自动备份数据库，保留30天               |
| 浏览器兼容 | Chrome / Edge 最新两个版本                 |
| 移动端适配 | 可考虑简化版移动H5，但首期建议PC优先       |
| 安全       | 所有操作需鉴权，防止SQL注入，飞书token校验 |
